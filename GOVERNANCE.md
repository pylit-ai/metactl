# Governance

`metactl` starts under single-maintainer project governance.

## Maintainer Role

Maintainers are responsible for:

- release decisions;
- compatibility policy;
- security response coordination;
- reviewing adapter and starter-library changes;
- enforcing public repository hygiene.

## Contribution Policy

Contributions require Developer Certificate of Origin sign-off. Maintainers may reject changes that weaken deterministic behavior, public boundary checks, or security posture.

## Compatibility Policy

Compatibility claims are tied to `docs/support-matrix.md` and `docs/conformance.md`. A target is compatibility-covered only when fixtures and CI checks exercise the target surface.

## Governance Changes

Governance can move to a broader model after independent users or implementers exist. Until then, maintainers should prefer clear review rules over committees.
